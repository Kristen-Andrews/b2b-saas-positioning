## 3. Primary alternative

**Their current workflow:**
Email inbox for invoices, manual entry into QuickBooks or NetSuite with project code lookup in a separate spreadsheet, physical or DocuSign approval routing.

**Why it persists:**
Bill.com and generic AP tools don't map cleanly to project accounting. Every services firm ends up with a workaround. The workaround becomes the process.

**What breaks first when they scale:**
The Controller can't tell the CFO which projects are actually profitable until 30 days after month-end, which means every quarterly board meeting starts with stale data.

## 6. Enemy

**The old way:**
Services firms treat AP as a procurement problem, so they buy generic AP automation, which forces them to bolt project accounting on top with spreadsheets and workarounds.

**Why it's broken now:**
Post-2024, private equity is buying services firms at record volume, and the first thing PE demands is project-level margin visibility. Firms with a 30-day lag between invoice and project code can't produce it.

**The cost of staying stuck:**
Project margin surprises at month-end. Board meetings that start with "we thought Project X was profitable, but..." Missed EBITDA targets. Failed PE diligence.

**The new way:**
Services-Native AP Automation treats every vendor invoice as a project accounting event first and a procurement event second.
